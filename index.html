Introduction

This project was inspired by Test First and their approach to teaching beginners to code with a test driven mindeset. After not seeing any comparable Python projects, I decided to start my own. Many thanks to Sarah Allen and Alex Chaffee for providing excellent beginner resources for learning Ruby and Javascript.

This tutorial is written entirely with Python 3 in mind. While there are still some teams and projects on Python 2, Python 2 reaches its end of life in 2020. Python 3 is the future, so that's the version we're using.

This tutorial assumes that you know how to open a terminal window and use it to execute commands. If you need help, here is a quick guide.

Setup

Install Python
You will need Python, pip, and virtualenv for this Python tutorial.
If you're using a Windows environment, go here for excellent instructions on setting up your environment.
If you're using Mac OS X, go here for similarly excellent instructions.
For Linux, here are yoursimilarly excellent instructions.
Create your Virtualenv

Note: For more information on virtual environments, check this out.

Once you've gotten Python, pip, and virtualenv set up, you'll need to create your virtualenv.

First, enter your course directory.

cd test_driven_python

Then, create your venv

virtualenv venv

Next, activate your virtualenv

source venv/bin/activate

Install the project requirements

pip install -r requirements.txt

Now enter your first lesson

cd lessons
cd lesson_0_hello_world

Open that lesson's index.html file in a web browser and follow the instructions there to get started!

Requirements

This project requires two different external Python Packages.

ipython is a command line alternative to the builtin Python shell. It has nice color formatting, and clearly labeled inputs and outputs. This might not mean a lot now, but will help you out when you're developing Python programs locally.

pytest is the testing package we'll use for this project. It is not Python's built in test framework, but it's much more readable and accessible, so that's what we're going to use.

Next Steps

Lessons live in the lessons directory in this project. Each lesson contains a test file and an `index.html` file. The html file leads to the web page for the lesson outline. You can also find this file in the dropdown `Lessons` menu on the nav bar above.

You should start with the first lesson (lesson_0_hello_world) to get a feel for how this course works. If you've finished that, fantastic! You'll now proceed through the lessons by opening the index.html file in each directory and following the instructions there.

Need Help?

Where do you go if something doesn't seem to be working?
  • Read the error message. Error messages are built to be informative, not scary. There's probably some infomration there that will lead you to your next step.
  • Google! Googling your error message is one of the most valuable skills you can develop in the early stages of learning to program.
  • Stack Overflow is a great resource for debugging your code.

  • iPython. iPython is an interactive shell where you can test out your code in real time. I've included it in the requirements file, so if you've set up your venv and installed requirements.txt, all you have to do is type:

    ipython

    in your command line with your venv activated, and you can try your code out. Never underestimate the power of debugging using well placed print statements!

  • Still can't find anything? Is something wrong with this Python course? If you think that's the case, hop over to the github repo and create an issue. Please provide instructions for how to reproduce the issue you're seeing.
